# Security Policy

## Supported versions

Security fixes are applied on a best-effort basis to the default branch.

## Reporting a vulnerability

Please do not open a public GitHub issue for suspected vulnerabilities.

Instead, contact the maintainer privately through the repository hosting platform's security reporting mechanism or private contact channel if one is available.

When reporting an issue, please include:

- A description of the vulnerability
- Steps to reproduce or validate the issue
- Potential impact
- Any suggested remediation or context

You can expect an acknowledgment within a reasonable timeframe, followed by triage and remediation updates when available.
